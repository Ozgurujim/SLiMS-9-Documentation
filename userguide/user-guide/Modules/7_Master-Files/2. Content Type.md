#### This sub-menu is used to manage the Content Type authority file .

**RDA Content Type definition:** defines the fundamental form of communication and the human sense used to perceive the content in a resource.

**Examples:** text, cartographic image, performed music, still image.

![content_type](C:\Users\jr\Desktop\slims972-work\slims9documentation-userguide\user-guide\Modules\7_Masterfiles\content_type.png)



SLiMS comes with standard authoritative codes and Content Type names. It is **strongly recommended** that you only edit this master file if there has been a change in the official standards [ see https://www.loc.gov/standards/valuelist/rdacontent.html ] . Do not delete entries.

If you wish to edit an entry you must select it , click the little edit pen button, and then on the resulting screen also click the EDIT button to enable editing. It's a type of "safety mechanism".

SLiMS does not translate master file entries. If you do not choose to use English terms,  you should edit the Content Type **Name** in the master-file to the equivalent term in your preferred language. Do not alter the  three-character codes.

The layout and function of this module's interface is similar to other master-file entry/management screens.
